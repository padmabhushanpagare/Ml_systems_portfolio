● Executive Summary

We replaced a brittle rule-based heuristic system with an XGBoost regression pipeline to tackle inaccurate delivery times. By engineering geospatial clusters and rush-hour density features, we reduced the Mean Absolute Error (MAE) from ~8 mins to 4.44 mins. The final model achieved an R² of 0.63, directly improving fleet utilization and reducing operational stress.

Business Problem

Inaccurate ETAs kill retention. Overestimates lead to cart abandonment; underestimates cause customer frustration and support tickets. The goal was simple but difficult: build a system that digests real-time signals (weather, traffic, kitchen load) and outputs an accurate prediction within milliseconds, without breaking the checkout flow.

Data Overview

We utilized a dataset of ~45,000 historical deliveries. The data included noisy GPS logs, timestamped status updates, and categorical metadata about drivers and restaurants.

Static Features

  • • Pickup/Dropoff Lat/Lng
  • • Restaurant Category & Avg Prep
  • • Driver Rating (1-5 scale)

Dynamic Signals

  • • Road Traffic Density
  • • Weather Conditions (Rain/Fog)
  • • Current Active Order Count

Feature Engineering Strategy

Raw coordinates and timestamps aren't predictive enough on their own. We engineered specific features to capture the nuances of urban logistics:

  • Geospatial Clustering: We applied K-Means clustering to raw coordinates to create "traffic zones". This allowed the model to learn neighborhood-specific congestion patterns (e.g., downtown vs. suburbs).
  • Rush Hour Flags: Boolean features for 8-10am and 5-7pm windows based on local time.
  • Distance Haversine: Calculated the precise Great Circle distance between pickup and dropoff points.

Modeling Approach

We benchmarked multiple regressors to establish a solid baseline. Linear Regression struggled with non-linear relationships (R²: 0.35). Random Forest improved performance but was too slow for real-time inference.

XGBoost was the clear winner. It effectively handled the non-linear interactions between weather, time of day, and traffic clusters. We optimized hyperparameters (learning rate, max depth, subsample) using 5-fold cross-validation to prevent overfitting.

Model Interpretability

We used SHAP (SHapley Additive exPlanations) to explain model decisions to stakeholders. The analysis revealed that while 'Distance' was the primary driver, 'Restaurant Prep Time' variance was a massive hidden bottleneck, often outweighing traffic conditions for short trips. This insight led to a separate operational initiative to help restaurants optimize their kitchen workflows.

Production Considerations

Deploying this required a robust MLOps strategy:

  • Feature Store: We used Redis to serve pre-computed features (like cluster IDs and driver stats) with sub-5ms latency.
  • Drift Monitoring: We set up alerts for input drift. If "Traffic Density" distribution shifts (e.g., due to a major event), the system can fallback to a simpler heuristic model.
  • Retraining: Automated pipelines retrain the model weekly to adapt to seasonal changes in traffic and demand.
